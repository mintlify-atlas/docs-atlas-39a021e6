## Overview

Justina Core Backend is fully containerized and ready for Docker deployment. This guide covers building Docker images, running containers, and deploying with Docker Compose.

## Dockerfile

The backend includes a multi-stage Dockerfile optimized for production deployment:

```dockerfile
# Imagen base con Java 21
FROM eclipse-temurin:21-jdk

# Directorio de trabajo dentro del contenedor
WORKDIR /app

COPY . .

# Dar permisos al wrapper de Maven
RUN chmod +x mvnw

# Construir la aplicación
RUN ./mvnw clean package -DskipTests

# Exponer el puerto
EXPOSE 8080

# Comando para iniciar la app
CMD ["sh", "-c", "java -jar target/*.jar --server.port=${PORT}"]
```

### Dockerfile Features

- **Base Image**: Eclipse Temurin JDK 21 (official OpenJDK distribution)
- **Build Process**: Uses Maven wrapper to compile application inside container
- **Port**: Exposes port 8080 (configurable via `PORT` environment variable)
- **Startup**: Dynamic JAR execution with environment-based port configuration

## Building the Docker Image

<Steps>
  <Step title="Navigate to Backend Directory">
    ```bash
    cd backend
    ```
  </Step>

  <Step title="Build Docker Image">
    ```bash
    docker build -t justina-backend:latest .
    ```

    This command:
    - Builds the image using the Dockerfile
    - Tags it as `justina-backend:latest`
    - Compiles the Spring Boot application inside the container
  </Step>

  <Step title="Verify Image Creation">
    ```bash
    docker images | grep justina-backend
    ```

    Expected output:
    ```
    justina-backend   latest   abc123def456   2 minutes ago   500MB
    ```
  </Step>
</Steps>

## Running the Container

### Basic Execution

```bash
docker run -p 8080:8080 justina-backend:latest
```

<Warning>
  This basic execution uses H2 in-memory database. Data will be lost when the container stops. Use environment variables to configure PostgreSQL for persistence.
</Warning>

### Production Execution with PostgreSQL

```bash
docker run -p 8080:8080 \
  -e JWT_SECRET_KEY="your-secure-secret-key-min-256-bits" \
  -e SPRING_DATASOURCE_URL="jdbc:postgresql://postgres-host:5432/justina" \
  -e SPRING_DATASOURCE_USERNAME="postgres" \
  -e SPRING_DATASOURCE_PASSWORD="secure-password" \
  -e SPRING_DATASOURCE_DRIVER_CLASS_NAME="org.postgresql.Driver" \
  -e SPRING_JPA_HIBERNATE_DDL_AUTO="update" \
  -e SPRING_JPA_SHOW_SQL="false" \
  -e SPRING_JPA_PROPERTIES_HIBERNATE_DIALECT="org.hibernate.dialect.PostgreSQLDialect" \
  justina-backend:latest
```

### Running in Background (Detached Mode)

```bash
docker run -d \
  --name justina-backend \
  -p 8080:8080 \
  -e JWT_SECRET_KEY="your-secret-key" \
  justina-backend:latest
```

## Docker Compose Deployment (Recommended)

Docker Compose simplifies multi-container deployments by orchestrating the backend and PostgreSQL database.

### Complete docker-compose.yml

Create a `docker-compose.yml` file in your project root:

<CodeGroup>

```yaml docker-compose.yml
version: '3.8'

services:
  justina-backend:
    build: ./backend
    container_name: justina-backend
    ports:
      - "8080:8080"
    environment:
      - PORT=8080
      - JWT_SECRET_KEY=${JWT_SECRET_KEY}
      - SPRING_DATASOURCE_URL=jdbc:postgresql://postgres:5432/justina
      - SPRING_DATASOURCE_USERNAME=postgres
      - SPRING_DATASOURCE_PASSWORD=${POSTGRES_PASSWORD}
      - SPRING_DATASOURCE_DRIVER_CLASS_NAME=org.postgresql.Driver
      - SPRING_JPA_HIBERNATE_DDL_AUTO=update
      - SPRING_JPA_SHOW_SQL=false
      - SPRING_JPA_PROPERTIES_HIBERNATE_DIALECT=org.hibernate.dialect.PostgreSQLDialect
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - justina-network

  postgres:
    image: postgres:15-alpine
    container_name: justina-postgres
    environment:
      - POSTGRES_DB=justina
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres-data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped
    networks:
      - justina-network

volumes:
  postgres-data:
    driver: local

networks:
  justina-network:
    driver: bridge
```

```env .env
# Security
JWT_SECRET_KEY=your-super-secure-secret-key-must-be-at-least-256-bits-long

# Database
POSTGRES_PASSWORD=your-secure-postgres-password
```

</CodeGroup>

### Compose Features Explained

| Feature | Description |
|---------|-------------|
| **Health Checks** | PostgreSQL container includes health check to ensure database is ready before backend starts |
| **Dependencies** | Backend waits for PostgreSQL to be healthy before starting |
| **Volumes** | Persistent data storage for PostgreSQL database |
| **Networks** | Isolated bridge network for inter-container communication |
| **Restart Policy** | Automatic restart on failure (unless manually stopped) |
| **Environment Variables** | Loaded from `.env` file for secure configuration |

### Starting the Application

<Steps>
  <Step title="Create Environment File">
    Create a `.env` file in the same directory as `docker-compose.yml`:

    ```bash
    # .env
    JWT_SECRET_KEY=your-super-secure-secret-key-must-be-at-least-256-bits-long
    POSTGRES_PASSWORD=your-secure-postgres-password
    ```

    <Warning>
      Never commit `.env` files to version control. Add it to `.gitignore`.
    </Warning>
  </Step>

  <Step title="Start Services">
    ```bash
    docker-compose up -d
    ```

    This command:
    - Builds the backend image (if not already built)
    - Pulls PostgreSQL 15 image
    - Creates volumes and networks
    - Starts both containers in detached mode
  </Step>

  <Step title="Verify Deployment">
    Check container status:
    ```bash
    docker-compose ps
    ```

    Expected output:
    ```
    NAME                COMMAND                  STATUS         PORTS
    justina-backend     "sh -c 'java -jar ta…"   Up 30 seconds  0.0.0.0:8080->8080/tcp
    justina-postgres    "docker-entrypoint.s…"   Up 30 seconds  0.0.0.0:5432->5432/tcp
    ```
  </Step>

  <Step title="View Logs">
    ```bash
    # View all logs
    docker-compose logs -f

    # View backend logs only
    docker-compose logs -f justina-backend

    # View PostgreSQL logs only
    docker-compose logs -f postgres
    ```
  </Step>

  <Step title="Test API">
    ```bash
    curl http://localhost:8080/swagger-ui/index.html
    ```
  </Step>
</Steps>

### Stopping the Application

```bash
# Stop containers (preserve data)
docker-compose stop

# Stop and remove containers (preserve volumes)
docker-compose down

# Stop, remove containers and volumes (delete all data)
docker-compose down -v
```

## Docker Commands Reference

### Container Management

<CodeGroup>

```bash Start
# Start all services
docker-compose up -d

docker-compose up -d justina-backend
```

```bash Stop

docker-compose stop

```

```bash Restart

docker-compose restart

# Restart specific service
docker-compose restart justina-backend
```

```bash Remove
# Remove containers
docker-compose down

# Remove containers and volumes
docker-compose down -v

# Remove containers, volumes, and images
docker-compose down -v --rmi all
```

</CodeGroup>

### Logs and Debugging

```bash
# View logs
docker-compose logs -f

# View last 100 lines
docker-compose logs --tail=100 justina-backend

# Execute shell in running container
docker-compose exec justina-backend bash

# Execute psql in PostgreSQL container
docker-compose exec postgres psql -U postgres -d justina
```

### Image Management

```bash
# Rebuild image
docker-compose build justina-backend

# Rebuild without cache
docker-compose build --no-cache justina-backend

# Pull latest images
docker-compose pull
```

## Production Optimization

### Multi-Stage Dockerfile (Optimized)

For production, consider this optimized multi-stage Dockerfile:

```dockerfile
# Stage 1: Build
FROM eclipse-temurin:21-jdk AS builder
WORKDIR /app
COPY . .
RUN chmod +x mvnw
RUN ./mvnw clean package -DskipTests

# Stage 2: Runtime
FROM eclipse-temurin:21-jre
WORKDIR /app

# Create non-root user
RUN groupadd -r justina && useradd -r -g justina justina

# Copy JAR from builder
COPY --from=builder /app/target/*.jar app.jar

# Change ownership
RUN chown -R justina:justina /app

# Switch to non-root user
USER justina

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Benefits

- **Smaller Image**: Uses JRE instead of full JDK (~200MB reduction)
- **Security**: Runs as non-root user
- **Layer Caching**: Separates build and runtime stages
- **Build Artifacts**: Only includes compiled JAR, not source code

## Health Checks

Add health check endpoint monitoring:

```yaml
services:
  justina-backend:
    # ... other config
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

<Warning>
  Ensure Spring Boot Actuator is enabled in your application for health check endpoints.
</Warning>
