## Overview

The AI analysis pipeline processes surgical trajectory data through five sequential steps to generate a comprehensive performance score and actionable feedback. The pipeline is implemented in `analysis_pipeline.py` and analyzes biomechanical metrics, pattern adherence, and risk factors.

## Pipeline Architecture

```python
def run_pipeline(trajectory_data: Dict) -> Tuple[float, str]:
    # PASO 1: Ingesta y Limpieza
    df = _paso1_ingestar_y_limpiar(trajectory_data)
    
    # PASO 2: Métricas de Destreza (Física)
    metricas = _paso2_calcular_destreza(df)
    
    # PASO 3: Comparación con Patrón Oro
    benchmarking = _paso3_benchmarking(df)
    
    # PASO 4: Análisis de Riesgo
    riesgo = _paso4_analizar_riesgo(df)
    
    # PASO 5: Feedback Inteligente
    score, feedback = _paso5_generar_feedback(metricas, benchmarking, riesgo)
    
    return score, feedback
```

### Processing Logic

```python
def _paso1_ingestar_y_limpiar(trajectory_data: Dict) -> pd.DataFrame:
    movements = trajectory_data["movements"]
    df = pd.DataFrame([{
        "x": m["coordinates"][0],
        "y": m["coordinates"][1],
        "z": m["coordinates"][2] if len(m["coordinates"]) > 2 else 0,
        "event": m["event"],
        "timestamp": m["timestamp"]
    } for m in movements])
    
    df = df.sort_values("timestamp").reset_index(drop=True)
    # Tiempo relativo en segundos
    df["t"] = (df["timestamp"] - df["timestamp"].iloc[0]) / 1000.0
    df["dt"] = df["t"].diff().fillna(0)
    
    return df
```

**Output**: Pandas DataFrame with columns `[x, y, z, event, timestamp, t, dt]`

## Step 2: Dexterity Metrics (Biomechanics)

**Purpose**: Calculate physical performance indicators using calculus-based motion analysis.

### Calculated Metrics

<AccordionGroup>
  <Accordion title="Velocity (v)" icon="gauge">
    Instantaneous speed of instrument movement:
    
    ```python
    # Distance between consecutive points
    dx = df["x"].diff().fillna(0)
    dy = df["y"].diff().fillna(0)
    dz = df["z"].diff().fillna(0)
    dist = np.sqrt(dx**2 + dy**2 + dz**2)
    
    # Velocity = distance / time
    v = dist / df["dt"].replace(0, np.inf)
    v = v.replace(np.inf, 0)
    ```
    
    **Interpretation**: Higher average velocity may indicate rushed movements.
  </Accordion>
  
  <Accordion title="Acceleration (a)" icon="rocket">
    Rate of velocity change:
    
    ```python
    # a = dv/dt
    a = v.diff() / df["dt"].replace(0, np.inf)
    a = a.replace(np.inf, 0)
    ```
    
    **Interpretation**: High max acceleration suggests jerky, uncontrolled movements.
  </Accordion>
  
  <Accordion title="Jerk (j) - Smoothness" icon="wave-pulse">
    Rate of acceleration change (third derivative of position):
    
    ```python
    # j = da/dt
    j = a.diff() / df["dt"].replace(0, np.inf)
    j = j.replace(np.inf, 0)
    j_avg = j.abs().mean()
    ```
    
    **Interpretation**: Lower jerk = smoother, more controlled movements. This is the gold standard metric for surgical skill assessment.
  </Accordion>
  
  <Accordion title="Economy of Motion" icon="route">
    Efficiency ratio comparing actual path to optimal straight line:
    
    ```python
    total_dist = dist.sum()  # Total path length
    p1 = np.array([df["x"].iloc[0], df["y"].iloc[0], df["z"].iloc[0]])
    p2 = np.array([df["x"].iloc[-1], df["y"].iloc[-1], df["z"].iloc[-1]])
    direct_dist = np.linalg.norm(p2 - p1)  # Straight-line distance
    
    economia = total_dist / direct_dist if direct_dist > 0 else 1.0
    ```
    
    **Interpretation**: Ideal value is close to 1.0. Values > 1.8 indicate excessive meandering.
  </Accordion>
</AccordionGroup>

## Step 3: Gold Pattern Comparison

**Purpose**: Benchmark the actual trajectory against an ideal straight-line path.

### Algorithm

```python
def _paso3_benchmarking(df: pd.DataFrame) -> Dict:
    p_start = np.array([df["x"].iloc[0], df["y"].iloc[0], df["z"].iloc[0]])
    p_end = np.array([df["x"].iloc[-1], df["y"].iloc[-1], df["z"].iloc[-1]])
    
    def dist_to_line(p, a, b):
        """Calculate perpendicular distance from point to line segment"""
        if np.all(a == b): 
            return np.linalg.norm(p-a)
        return np.linalg.norm(np.cross(b-a, a-p)) / np.linalg.norm(b-a)
    
    # Calculate deviation for each trajectory point
    desviaciones = [
        dist_to_line(np.array([r.x, r.y, r.z]), p_start, p_end) 
        for r in df.itertuples()
    ]
    
    return {
        "desviacion_avg": np.mean(desviaciones),
        "precision": max(0, 100 - np.mean(desviaciones) * 10)
    }
```

**Metrics**:
- `desviacion_avg`: Average perpendicular distance from ideal path
- `precision`: Percentage score (100 = perfect adherence to straight line)

## Step 4: Risk Analysis

**Purpose**: Identify critical events and spatial patterns of complications.

### Event Counting

```python
def _paso4_analizar_riesgo(df: pd.DataFrame) -> Dict:
    tumor_removals = (df["event"] == "TUMOR_REMOVAL").sum()
    hemorrhages = (df["event"] == "HEMORRHAGE").sum()
    kidney_touches = (df["event"] == "KIDNEY_TOUCH").sum()
```

### Critical Event Types

| Event | Impact | Penalty |
|-------|--------|--------|
| `TUMOR_REMOVAL` | Positive - indicates progress | +0 (tracked for completion %) |
| `HEMORRHAGE` | Critical - bleeding event | -25 points |
| `KIDNEY_TOUCH` | Moderate - healthy tissue damage | -5 points |
| `TUMOR_TOUCH` | Neutral - expected interaction | 0 |

### Spatial Analysis

The algorithm identifies high-risk quadrants where complications occurred:

```python
# Calculate grid center
mid_x = (df["x"].max() + df["x"].min()) / 2
mid_y = (df["y"].max() + df["y"].min()) / 2

problemas = df[df["event"].isin(["KIDNEY_TOUCH", "HEMORRHAGE"])]
cuadrantes_criticos = []

for p in problemas.itertuples():
    pos = ""
    pos += "Sup" if p.y > mid_y else "Inf"
    pos += "-Der" if p.x > mid_x else "-Izq"
    if pos not in cuadrantes_criticos: 
        cuadrantes_criticos.append(pos)
```

**Output**: List of problematic quadrants (e.g., `["Sup-Der", "Inf-Izq"]`)

## Step 5: Intelligent Feedback Generation

**Purpose**: Synthesize all metrics into a final score and structured feedback report.

### Scoring Algorithm

```python
def _paso5_generar_feedback(m: Dict, b: Dict, r: Dict) -> Tuple[float, str]:
    score = 100.0
    
    # Penalties
    score -= r["kidney_touches"] * 5      # -5 per healthy tissue damage
    score -= r["hemorrhages"] * 25        # -25 per hemorrhage
    if m["economia"] > 1.8: 
        score -= 10                        # -10 for inefficient path
    
    # Completion bonus calculation
    progreso_tumor = (r["removals"] / 25.0) * 100  # Assuming 25 tumor fragments
    
    score = max(0, min(100, score))  # Clamp to [0, 100]
```

## Pipeline Invocation

The pipeline is called from the WebSocket client after retrieving trajectory data:

```python
# websocket_client.py
def procesar_cirugia_async(self, surgery_id):
    # 1. Get trajectory
    trajectory_data = self.client.get_trajectory(surgery_id)
    
    # 2. Run analysis pipeline
    score, feedback = run_pipeline(trajectory_data)
    
    # 3. Send results back
    self.client.send_analysis(surgery_id, score, feedback)
```

## Performance Considerations

- **Processing Time**: Typical analysis completes in 50-200ms for 500-2000 trajectory points
- **Memory Usage**: DataFrame operations are memory-efficient for datasets < 10MB
- **Concurrency**: Each surgery is processed in a separate thread to avoid blocking WebSocket listener

<Info>
The pipeline is designed to be stateless and can process surgeries in parallel if needed.
</Info>
