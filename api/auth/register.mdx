Creates a new user account in the Justina surgical simulation platform. The username must be unique and follow the validation rules specified below.

## Request Body

<ParamField body="username" type="string" required>
  Desired username for the new account
  
  - **Validation**: Required, 4-50 characters
  - **Must be unique**: Cannot match existing usernames
  - **Example**: `surgeon_master`
</ParamField>

<ParamField body="password" type="string" required>
  Password for the new account
  
  - **Validation**: Required, 6-100 characters
  - **Example**: `justina2024`
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming user registration
  
  **Example**: `Usuario registrado con éxito`
</ResponseField>

<RequestExample>

```bash cURL

  -H "Content-Type: application/json" \
  -d '{

    "password": "justina2024"
  }'
```

## Validation Rules

### Username
- Must not be blank
- Minimum length: 4 characters
- Maximum length: 50 characters
- Must be unique across all users

### Password
- Must not be blank
- Minimum length: 6 characters
- Maximum length: 100 characters
- Stored securely using encryption

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | User registered successfully |
| 400 | Invalid data, validation error, or user already exists |

## Next Steps

After successful registration, use the [login endpoint](/api/auth/login) to authenticate and receive a JWT token for API access.
