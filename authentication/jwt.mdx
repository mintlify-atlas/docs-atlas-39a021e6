## JWT Token Structure

Justina Core Backend uses the **auth0 JWT library** to create and validate JSON Web Tokens. Each token contains three parts: header, payload, and signature.

### Token Claims

Every JWT token issued by Justina includes the following claims:

| Claim | Description | Example |
|-------|-------------|----------|
| `iss` | Token issuer | `"Justina_Backend"` |
| `sub` | Subject (username) | `"surgeon_master"` |
| `userId` | Unique user identifier | `"a1b2c3d4-e5f6-7890-abcd-ef1234567890"` |
| `role` | User role | `"ROLE_SURGEON"` or `"ROLE_AI"` |
| `iat` | Issued at timestamp | `1709755200` |
| `exp` | Expiration timestamp | `1709841600` |

## Token Generation

Tokens are generated by the `JwtService` when users successfully log in.

```java
// From JwtService.java:20-31
public String createToken(UUID userId, String username, String role) {
    Algorithm algorithm = Algorithm.HMAC256(secretKey);

    return JWT.create()
            .withIssuer(issuer)
            .withSubject(username)
            .withClaim("userId", userId.toString())
            .withClaim("role", role)
            .withIssuedAt(new Date())
            .withExpiresAt(new Date(System.currentTimeMillis() + 86400000)) // 1 day
            .sign(algorithm);
}
```

### Token Signing

Tokens are signed using **HMAC-SHA256** algorithm with a secret key stored in environment variables:

```properties
# From application.properties
jwt.secret.key=${JWT_SECRET_KEY}
```

<Warning>
  The `JWT_SECRET_KEY` should be a strong, random string stored securely as an environment variable. Never commit this value to version control.
</Warning>

## Login & Token Retrieval

### Login Request

To obtain a JWT token, send a POST request to the login endpoint:

<CodeGroup>
```bash cURL
curl -X POST https://api.justina.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "surgeon_master",
    "password": "justina2024"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.justina.com/api/v1/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  credentials: 'include', // Important for cookies
  body: JSON.stringify({
    username: 'surgeon_master',
    password: 'justina2024',
  }),
});

const data = await response.json();
console.log(data.token); // JWT token
```

```python Python
import requests

response = requests.post(
    'https://api.justina.com/api/v1/auth/login',
    json={
        'username': 'surgeon_master',
        'password': 'justina2024'
    }
)

data = response.json()
token = data['token']
```

```java Java
LoginRequestDTO loginRequest = new LoginRequestDTO("surgeon_master", "justina2024");

ResponseEntity<?> response = authController.login(loginRequest, httpServletResponse);
Map<String, Object> body = (Map<String, Object>) response.getBody();
String token = (String) body.get("token");
```
</CodeGroup>

### Cookie-Based Authentication

In addition to the response body, the server also sets an HTTP-only cookie:

```java
// From AuthController.java:66-71
jakarta.servlet.http.Cookie cookie = new jakarta.servlet.http.Cookie("jwt-token", authResponse.token());
cookie.setHttpOnly(true);
cookie.setSecure(true);
cookie.setPath("/");
cookie.setMaxAge(86400); // 24 hours
cookie.setAttribute("SameSite", "None");
```

**Cookie Properties:**
- **Name**: `jwt-token`
- **HttpOnly**: `true` (prevents JavaScript access)
- **Secure**: `true` (only sent over HTTPS)
- **SameSite**: `None` (allows cross-origin requests)
- **MaxAge**: `86400` seconds (24 hours)

## Using JWT Tokens

Once you have a JWT token, include it in API requests using one of two methods:

### Method 1: Authorization Header (Recommended)

Include the token in the `Authorization` header with the `Bearer` scheme:

<CodeGroup>
```bash cURL
curl -X GET https://api.justina.com/api/v1/surgeries \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```javascript JavaScript
const response = await fetch('https://api.justina.com/api/v1/surgeries', {
  headers: {
    'Authorization': `Bearer ${token}`,
  },
});
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {token}'
}

response = requests.get(
    'https://api.justina.com/api/v1/surgeries',
    headers=headers
)
```

```java Java
HttpHeaders headers = new HttpHeaders();
headers.setBearerAuth(token);

HttpEntity<?> entity = new HttpEntity<>(headers);
ResponseEntity<String> response = restTemplate.exchange(
    "https://api.justina.com/api/v1/surgeries",
    HttpMethod.GET,
    entity,
    String.class
);
```
</CodeGroup>

### Method 2: Cookie (Automatic for Browsers)

If you logged in via a browser, the `jwt-token` cookie is automatically included in requests:

```javascript
// No need to manually set Authorization header
const response = await fetch('https://api.justina.com/api/v1/surgeries', {
  credentials: 'include', // Ensures cookies are sent
});
```

## Token Validation

The `JwtAuthenticationFilter` validates tokens on every request:

<Steps>
  <Step title="Extract Token">
    The filter first checks for the `jwt-token` cookie, then falls back to the `Authorization` header.
  </Step>
  
  <Step title="Decode & Verify">
    The token is decoded and verified using the same HMAC-SHA256 secret key.
  </Step>
  
  <Step title="Extract Claims">
    The username, user ID, and role are extracted from the token claims.
  </Step>
  
  <Step title="Load User Details">
    The user is loaded from the database to ensure they still exist.
  </Step>
  
  <Step title="Set Security Context">
    If valid, the user's authentication is set in the Spring Security context.
  </Step>
</Steps>

```java
// From JwtAuthenticationFilter.java:96-111
try {
    String usernameFromToken = jwtService.extractUsername(jwt);

    if (usernameFromToken != null && SecurityContextHolder.getContext().getAuthentication() == null) {
        UserDetails userDetails = this.userDetailsService.loadUserByUsername(usernameFromToken);

        if (jwtService.isTokenValid(jwt, userDetails)) {
            UsernamePasswordAuthenticationToken authToken = new UsernamePasswordAuthenticationToken(
                    userDetails, null, userDetails.getAuthorities());

            authToken.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
            SecurityContextHolder.getContext().setAuthentication(authToken);
        }
    }
} catch (Exception e) {
    // Token is invalid - request will be unauthorized
}
```

## Token Validation Methods

The `JwtService` provides several methods for extracting and validating token claims:

```java
// Extract username from token
public String extractUsername(String token)

// Extract user ID from token
public UUID extractUserId(String token)

// Extract role from token
public String extractRole(String token)

// Validate token against user details
public boolean isTokenValid(String jwt, UserDetails userDetails)
```

## Error Handling

### Invalid or Expired Tokens

If a token is invalid, expired, or missing, the API returns:

```json
{
  "error": "Unauthorized",
  "message": "Full authentication is required to access this resource",
  "status": 401
}
```

### Invalid Credentials

If login credentials are incorrect:

```json
{
  "error": "Credenciales inválidas"
}
```

<CodeGroup>

### Registration Response

Successful registration:

```json
{
  "message": "Usuario registrado con éxito"
}
```

User already exists:

```json

}
```

<Warning>
  Registration requires:
  - Username: 4-50 characters
  - Password: 6-100 characters
  
  All registered users are automatically assigned the `ROLE_SURGEON` role.
</Warning>

## Best Practices

1. **Store Tokens Securely**: Never store JWT tokens in localStorage on the browser. Use httpOnly cookies or secure session storage.

2. **Always Use HTTPS**: JWT tokens should only be transmitted over secure HTTPS connections.

3. **Handle Token Expiration**: Implement token refresh logic or prompt users to log in again after 24 hours.

4. **Validate on Every Request**: The server validates tokens on every protected endpoint to ensure security.

5. **Don't Include Sensitive Data**: JWT tokens are base64-encoded, not encrypted. Don't include sensitive information in token claims.
